# AFOQT Study

Single-user, client-only study site. No build step, no backend.

## Run locally
```
cd afoqt-site
python3 -m http.server 8080
# open http://localhost:8080
```

## Anthropic API key (required for daily content) — Claude Opus 4.6
Paste your Anthropic key in `#settings`. Each day the site calls Claude once and gets:

- **5 Greek/Latin root words** with origin, meaning, 3 English derivatives, mnemonic sentence, and an image-generation prompt
- **5 AFOQT drills** spanning all five question types
- **1 motivational quote**

One call per day; cached under today's date in `localStorage`. Old entries auto-pruned after 14 days. Recent roots are tracked so tomorrow's lesson avoids repeats. Without a key, 5 seed roots rotate deterministically from `roots.js`.

Get a key at [console.anthropic.com](https://console.anthropic.com/). Cost ≈ 1–2 cents/day.

## Root word icons (no extra key needed)
Each root gets a minimalist gold-on-navy SVG icon generated by Claude in the same daily call — no separate image API, no extra cost. Claude writes raw SVG markup per the `svg` field in the schema. The SVG is sanitized (scripts, event handlers, and external references stripped) and injected inline. Icons inherit the card's theme color via `stroke="currentColor"`.

Fallback: if Claude returns no valid SVG for a root, a gold monogram letter renders instead.

## Data & cloud sync

**Manual backup**
- Export: `#settings` → Export JSON (downloads `afoqt-backup-YYYY-MM-DD.json`)
- Import: `#settings` → Import JSON (replaces current state)
- Reset: `#settings` → Reset all data

**Automatic cross-device sync (private GitHub Gist)**
- Create a Personal Access Token at `github.com/settings/tokens` with scope `gist` only.
- Paste it into `#settings` → Cloud sync → Save.
- First sync auto-creates a private gist named `afoqt-state.json` and stores its ID locally.
- After that, every change auto-pushes ~20s after it happens. Page load pulls the remote if it's newer.
- Same token + same gist ID on another device = same progress everywhere.
- Conflict rule: "last-write-wins" by `lastModifiedAt` timestamp embedded in state.
- Your gist is listed at `gist.github.com/{your-username}`. Never shared; only you can read it.
